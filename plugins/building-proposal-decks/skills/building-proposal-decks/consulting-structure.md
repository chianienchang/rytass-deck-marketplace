# 顧問級簡報架構指南

做提案簡報前，先用這裡的原則幫使用者**規劃整份簡報的邏輯與頁序**，再開始套版型。目標：讓決策者能在最短時間內看懂、信任、並做出決定。

## 三個核心原則

1. **結論先行（金字塔原則）**
   先給結論／建議，再給支持理由，最後才是細節。決策者時間有限，不要讓他看到最後一頁才知道你要他做什麼。

2. **單一維度決策**
   多案並陳時，讓所有方案的「結構、流程、頁數」完全一致，只在「一個維度」上有差異（例如只差色彩風格）。這樣客戶不必比較一堆變因，只需在一條軸線上選擇——這是顧問讓客戶快速拍板的關鍵手法。

3. **每一頁只講一件事**
   一頁一個重點。資訊太多就拆頁。寧可多一頁，不要塞爆一頁。

## 提案敘事弧線（預設頁序）

適用於「設計提案 / 方案比較」這類簡報：

| 段落 | 頁型 | 作用 |
|------|------|------|
| 開場 | 封面 | 專案、主題、給誰看 |
| 全局 | 總覽（三卡） | 先讓人看到「有哪幾個方案、各自定位」 |
| 逐案展開 | 設計理念 → 展示頁 ×N | 每個方案先講理念，再看實際畫面 |
| 並排比較 | 對照表 | 把所有方案放在同一張表，逐維度比較 |
| 同步對照 | Recap 同步捲動 | 同一畫面三案並列，眼見為憑 |
| 收尾 | 建議與下一步 | 給建議、列出後續行動與時程 |

> 逐案展開時，**每個方案的頁型順序要一致**（理念 → 桌機 → 手機平板 → 流程…），呼應「單一維度決策」。

## 其他主題的建議頁序

- **策略／營運建議簡報**：封面 → 現況與痛點 → 建議方案（結論先行）→ 理由與佐證 → 風險與對策 → 下一步。可用「設計理念」頁型承載論點、「對照表」比較選項、「結尾」頁收行動。
- **數據／成效彙報**：封面 → 重點摘要（結論先行）→ 各項指標 → 洞察與解讀 → 建議。
- 沒有裝置畫面時，就少用「展示頁」，多用「設計理念」「對照表」「結尾」這幾種純內容頁型。

## SCQA 開場法（要更有說服力時）

在封面後、進入方案前，可加一頁鋪陳：
- **S**ituation 情境：現況是什麼
- **C**omplication 衝突：出現什麼問題／需求
- **Q**uestion 問題：所以我們要回答什麼
- **A**nswer 解答：這份提案的核心主張

## 規劃時要先問使用者的事

1. 這份簡報**給誰看**？（決策者層級、是否懂技術）
2. 你**希望對方看完做什麼決定**？（這決定了結尾頁要寫什麼）
3. 有**幾個方案**要並陳？差異維度是什麼？
4. 有沒有**現成畫面／截圖**要放？放哪些頁？
5. 大概要**幾頁**、什麼場合報告？

把答案先整理成一份頁序清單，跟使用者確認後再開始做。
